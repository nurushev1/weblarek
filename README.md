# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные
В данном проекте используются два типа данных: **покупатель** и **продукт**.

### Покупатель

Покупатель содержит следующие поля:
- `payment` - способ оплаты (`TPayment`);
- `address` - адрес доставки;
- `email` - электронная почта;
- `phone` - телефон.

```ts
type TPayment = 'card' | 'cash' | '';

interface IBuyer {
    payment: TPayment;
    address: string;
    email: string;
    phone: string;
}
```

### Продукт

Продукт содержит следующие поля:
- `id` - уникальный идентификатор товара;
- `title` - название товара;
- `image` - картинка товара;
- `category` - категория товара;
- `price` - цена товара;
- `description` - описание товара;

```ts
interface IProduct {
    id: string;
    title: string;
    image: string;
    category: string;
    price: number | null;
    description: string;
}
```

Назначение интерфейсов: 
Интерфейсы обеспечивают единую схему данных, которой придерживаются все модули проекта. 
Это позволяет:
- избежать ошибок при работе с данными;
- обеспечить строгую типизацию;
- облегчить взаимодействие между классами, компонентами и API.

## Модели данных

### Класс ProductCatalog
Назначение:
Отвечает за хранение и управление списком товаров, полученных с сервера, а также за выбранный товар для детального просмотра.

Конструктор:
`constructor(events: IEvents)`

Поля:
- `items: IProduct[]` — хранит массив всех товаров каталога;
- `selected: IProduct | null` — хранит выбранный товар для просмотра;
- `events: IEvents` — брокер событий для уведомления об изменениях данных;

Методы:
- `setItems(items: IProduct[]): void` - Сохраняет список товаров (создаётся копия массива через slice()) и генерирует событие `catalog:change`.
- `getItems(): IProduct[]` - Возвращает текущий список товаров.
- `getItem(id: string): IProduct | undefined` - Возвращает товар по его id, если он найден.
- `selectItem(item: IProduct): void` - Сохраняет выбранный товар и генерирует событие `catalog:item-selected`.
- `getSelected(): IProduct | null` - Возвращает текущий выбранный товар.

События:
- `catalog:change`
Данные: { items: IProduct[] }
Генерируется при обновлении списка товаров.
- `catalog:item-selected`
Данные: IProduct
Генерируется при выборе товара для просмотра.

### Класс Cart
Назначение:
Отвечает за хранение выбранных товаров и управление этой корзиной.

Конструктор:
`constructor(events: IEvents)`

Поля:
- `items: IProduct[]` — массив товаров, добавленных в корзину;
- `events: IEvents` — брокер событий для уведомления об изменениях корзины;

Методы:
- `getItems(): IProduct[]` — возвращает массив товаров в корзине;
- `addItem(product: IProduct)` — добавляет товар в корзину и генерирует событие `cart:change`;
- `removeItem(product: IProduct)` — удаляет товар из корзины и генерирует событие `cart:change`;
- `clearCart()` — очищает корзину и генерирует событие `cart:change`;
- `getTotal(): number` — возвращает общую стоимость товаров;
- `getCount(): number` — возвращает количество товаров в корзине;
- `contains(productId: string): boolean` — проверяет наличие товара в корзине;

События:
`cart:change`
Данные: { items: IProduct[] }
Генерируется при любом изменении корзины (добавление товара, удаление товара, очистка корзины).

### Класс Buyer
Назначение:
Отвечает за хранение и управление данными покупателя при оформлении заказа, а также их валидацию.

Конструктор:
constructor(events: IEvents)

Поля:
- `paymentMethod: TPayment` — выбранный способ оплаты;
- `address: string` — адрес доставки;
- `phone: string` — телефон покупателя;
- `email: string` — email покупателя;
- `events: IEvents` — брокер событий;

Методы:
- `setPaymentMethod(payment: TPayment)` — устанавливает способ оплаты и генерирует событие `buyer:changed`;
- `setAddress(address: string)` — устанавливает адрес и генерирует событие `buyer:changed`;
- `setEmail(email: string)` — устанавливает email и генерирует событие `buyer:changed`;
- `setPhone(phone: string)` — устанавливает телефон и генерирует событие `buyer:changed`;
- `getBuyerData(): IBuyer` — возвращает все данные покупателя;
- `reset()` — очищает все данные покупателя и генерирует событие `buyer:changed`;
- `validate(): IErrors` — проверяет корректность данных и возвращает объект ошибок;

События:
`buyer:changed`
Данные: { field: string } или без данных (при reset)
Генерируется при изменении любого поля или сбросе данных покупателя.

## Представление
Особенности слоя View:
- Компоненты представления не хранят бизнес-данные;
- Данные передаются в компоненты через метод render;
- Компоненты не получают данные самостоятельно;
- Для обработки пользовательских действий используются:
 - колбэки (actions) — для локальных взаимодействий;
 - брокер событий (IEvents) — для глобальных событий;
- Все события обрабатываются на уровне презентера;
- Поиск DOM-элементов выполняется только внутри контейнера компонента;
- Для поиска элементов используется утилита ensureElement.

### Класс OrderForm
Назначение:
Форма оформления заказа: выбор способа оплаты и ввод адреса доставки.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля:
- `addressInput: HTMLInputElement` — поле ввода адреса доставки;
- `cardButton: HTMLButtonElement` — кнопка выбора оплаты картой;
- `cashButton: HTMLButtonElement` — кнопка выбора оплаты наличными;
- `events: IEvents` — брокер событий;

Методы:
- `setAddressValue(value: string): void` — устанавливает значение поля адреса доставки;
- `setPaymentMethod(status: TPayment): void` — обновляет состояние кнопок выбора способа оплаты (карта / наличные);

Переопределённые методы:
- `onSubmit(): void` — генерирует событие отправки формы оформления заказа;

События:
- `order:change`
Данные: { payment: TPayment } или { address: string }
Генерируется при изменении способа оплаты или адреса доставки.
- `order:submit`
Данные: —
Генерируется при отправке формы оформления заказа.

### Класс ContactsForm
Назначение:
Форма ввода контактных данных пользователя (email и телефон) при оформлении заказа.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля:
- `emailInput: HTMLInputElement` — поле ввода email;
- `phoneInput: HTMLInputElement` — поле ввода телефона;
- `events: IEvents` — брокер событий;

Методы:
- `setEmailValue(value: string): void` — устанавливает значение email в поле ввода;
- `setPhoneValue(value: string): void` — устанавливает значение телефона в поле ввода;

Переопределённые методы:
- `onSubmit(): void` — генерирует событие отправки формы контактных данных;

События:
- `contacts:change`
Данные: { email: string } или { phone: string }
Генерируется при изменении полей email или телефона.
- `contacts:submit`
Данные: —
Генерируется при отправке формы контактных данных.

### Класс Success
Назначение:
Отображает сообщение об успешном оформлении заказа и итоговую сумму списания.

Конструктор:
- `constructor(container: HTMLElement, events: IEvents)`

Поля:
- `title: HTMLElement` — заголовок окна успешного оформления заказа;
- `description: HTMLElement` — текст с информацией о списании суммы;
- `closeButton: HTMLButtonElement` — кнопка закрытия окна;
- `events: IEvents` — брокер событий;

Методы:
`setTotal(value: number): void` — устанавливает текст с суммой списания заказа;

События:
`success:close`
Данные: —
Генерируется при нажатии на кнопку закрытия окна успешного заказа.

Презентер
Назначение:
Презентер отвечает за связывание слоёв Model и View и реализует основную логику приложения.
В данном проекте презентер реализован в файле main.ts без выделения отдельного класса.

Обязанности презентера:
- подписка на события от моделей и представлений;
- вызов методов моделей при действиях пользователя;
- обновление представлений при изменении данных;
- управление отображением модальных окон.

Особенности:
- не генерирует события;
- не хранит данные;
- не содержит DOM-логики;
- работает только через модели и представления;
- обновляет интерфейс только при событиях от моделей или при открытии модальных окон.

### События приложения
## События моделей (Model)
### ProductCatalog
- `catalog:change`
Данные: { items: IProduct[] }
Описание: генерируется при обновлении списка товаров
- `catalog:item-selected`
Данные: IProduct
Описание: генерируется при выборе товара для просмотра

### Cart
- `cart:change`
Данные: { items: IProduct[] }
Описание: генерируется при любом изменении корзины (добавление / удаление / очистка)

### Buyer
- `buyer:changed`
Данные: { field: string } | undefined
Описание: генерируется при изменении любого поля покупателя или при сбросе данных

## События представления (View)

### ContactsForm

Данные: { email: string } | { phone: string }

- `contacts:submit`

### Modal

### Success
