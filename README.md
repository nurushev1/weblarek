# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные
В данном проекте используются два типа данных: **покупатель** и **продукт**.

### Покупатель

Покупатель содержит следующие поля:
- `payment` - способ оплаты (`TPayment`);
- `address` - адрес доставки;
- `email` - электронная почта;
- `phone` - телефон.

```ts
type TPayment = 'card' | 'cash' | '';

interface IBuyer {
    payment: TPayment;
    address: string;
    email: string;
    phone: string;
}
```

### Продукт

Продукт содержит следующие поля:
- `id` - уникальный идентификатор товара;
- `title` - название товара;
- `image` - картинка товара;
- `category` - категория товара;
- `price` - цена товара;
- `description` - описание товара;

```ts
interface IProduct {
    id: string;
    title: string;
    image: string;
    category: string;
    price: number | null;
    description: string;
}
```

Назначение интерфейсов: 
Интерфейсы обеспечивают единую схему данных, которой придерживаются все модули проекта. 
Это позволяет:
- избежать ошибок при работе с данными;
- обеспечить строгую типизацию;
- облегчить взаимодействие между классами, компонентами и API.

## Модели данных

### Класс ProductCatalog
Назначение:
Отвечает за хранение и управление списком товаров, полученных с сервера, а также работа с выбранным товаром.

Конструктор:
`constructor(events: IEvents)`

Поля:
- `products: IProduct[]` - хранит массив всех товаров;
- `selectedProduct: IProduct | null` - хранит товар, выбранный для подробного отображения;
- `events: IEvents` - брокер событий для уведомления об изменениях данных;

Методы:
- `setProducts(products: IProduct[])` - сохраняет массив товаров и генерирует событие `catalog:changed`;
- `getProducts(): IProduct[]` - возвращает массив товаров;
- `getProductById(id: string): IProduct | undefined` - возвращает товар по id;
- `setSelectedProduct(product: IProduct)` - сохраняет выбранный товар и генерирует событие `product:selected`;
- `getSelectedProduct(): IProduct | null` - возвращает выбранный товар.

События:
- `catalog:changed` - Данные `{ products: IProduct[] }`, генерируется при обновлении списка товаров
- `product:selected` - Данные `{ product: IProduct }`, генерируется при выборе товара для просмотра

### Класс Cart
Назначение:
Отвечает за хранение выбранных товаров и управление этой корзиной.

Конструктор:
`constructor(events: IEvents)`

Поля:
- `products: IProduct[]` -  массив выбранных товаров;
- `events: IEvents` — брокер событий для уведомления об изменениях корзины;

Методы:
- `getProducts(): IProduct[]` - возвращает массив выбранных товаров;
- `addProduct(product: IProduct)` - добавляет товар в корзину и генерирует событие `cart:changed`;
- `removeProduct(id: string)` - удаляет товар из корзины и генерирует событие `cart:changed`;
- `clearCart()` - очищает корзину и генерирует событие `cart:changed`;
- `getAllCost(): number` - возвращает общую стоимость товаров;
- `getQuantity(): number` - возвращает количество товаров;
- `hasProduct(id: string): boolean` - проверяет наличие товара в корзине.

События:
`cart:changed` - { products: IProduct[] }, генерируется при любом изменении содержимого корзины (добавление, удаление, очистка);

### Класс Buyer
Назначение:
Отвечает за сбор данных покупателей для оформления заказа.

Конструктор:
`constructor(events: IEvents)`

Поля:
- `payment: TPayment` - вид оплаты;
- `address: string` - адрес;
- `email: string` - электронная почта;
- `phone: string` - телефон покупателя;
- `events: IEvents` - брокер событий для уведомления об изменениях данных;

Методы:
- `setData(data: Partial<IBuyer>)` - сохраняет данные покупателя и генерирует событие `buyer:changed`;
- `getData(): IBuyer` - возвращает все данные покупателя;
- `clearData()` - очищает данные покупателя и генерирует событие `buyer:changed`;
- `validateData(): Partial<Record<keyof IBuyer, string>>` - валидирует данные и возвращает объект с ошибками;

События:
`buyer:changed` - IBuyer, генерируется при изменении или очистке данных покупателя;

### Слой коммуникации
Класс ServerConnector
Назначение:
Отвечает за взаимодействие с сервером. Получает данные товаров и отправляет данные покупателя. Не хранит данные, только передает их между API и моделями.

Конструктор:
`constructor(api: Api)`

Поля:
- `api: Api` - экземпляр Api, через который выполняются запросы;

Методы:
- `fetchProducts(): Promise<IProductsResponse>` - получает с сервера список товаров;
- `sendOrder(order: IOrderRequest): Promise<IOrderResponse>` - отправляет данные покупателя на сервер;

## Представление

Особенности слоя View:
- Компоненты представления не хранят бизнес-данные;
- Данные передаются в компоненты через метод render;
- Компоненты не получают данные самостоятельно;
- Для обработки пользовательских действий используются:
 - колбэки (actions) — для локальных взаимодействий;
 - брокер событий (IEvents) — для глобальных событий;
- Все события обрабатываются на уровне презентера;
- Поиск DOM-элементов выполняется только внутри контейнера компонента;
- Для поиска элементов используется утилита ensureElement.

### Класс Header
Назначение:
Отвечает за отображение шапки сайта и инициирует открытие корзины.

Конструктор:
`constructor(events: IEvents, container: HTMLElement)`

Поля:
- `counterElement: HTMLElement` — элемент счётчика;
- `basketButton: HTMLButtonElement` — кнопка корзины;
- `events: IEvents` — брокер событий;

Сеттеры:
- `counter: number` — обновляет отображение количества товаров;

События:
- `basket:open` — генерируется при клике;

Особенности:
- Использует ensureElement;
- Работает только внутри контейнера;
- Генерирует событие через IEvents.

### Класс Gallery
Назначение:
Отображает список карточек товаров.

Конструктор:
`constructor(container: HTMLElement)`

Поля:
- `catalogElement: HTMLElement` — контейнер каталога;

Сеттеры:
- `catalog: HTMLElement[]` — заменяет список элементов;

Особенности:
- Не генерирует событий;
- Только отображает переданные элементы;
- Не содержит логики.

### Класс Modal
Назначение:
Управляет отображением модального окна и его содержимым.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля:
- `contentElement: HTMLElement` — контейнер для отображаемого контента;
- `closeButton: HTMLButtonElement` — кнопка закрытия;
- `events: IEvents` — брокер событий;

Сеттеры:
- `content: HTMLElement` — устанавливает новый контент модального окна (заменяет текущее содержимое через replaceChildren)

Методы:
- `open()` — открывает модальное окно;
- `close()` — закрывает модальное окно, очищает контент и генерирует событие modal:close;
- `clear()` — очищает содержимое модального окна (contentElement.innerHTML = '');

События:
- `modal:close` — генерируется при закрытии (клик по крестику или overlay);

Особенности:
- Отображает любой UI-компонент;
- Закрывается по крестику и клику вне области контента (overlay);
- Использует ensureElement для поиска DOM-элементов;
- Не хранит состояние контента (каждый render заменяет DOM);
- Очистка содержимого выполняется через clear() при закрытии;
- Использует модификатор modal_active для отображения.

### Класс ProductCard
Назначение:
Базовый класс карточек товаров.

Конструктор:
`constructor(container: HTMLElement)`

Поля:
- `titleElement: HTMLElement` — DOM-элемент заголовка товара, отображает название;
- `priceElement: HTMLElement` — DOM-элемент цены товара, отображает стоимость или статус доступности;
- `imageElement: HTMLImageElement` — DOM-элемент изображения товара.

Сеттеры:
- `title: string` — устанавливает текст названия товара в titleElement;
- `price: number | null` — устанавливает цену товара;
- `image: string` — устанавливает изображение товара через setImage, используя imageElement.

Особенности:
- Абстрактный класс;
- Не генерирует события;
- Не содержит бизнес-логики и не хранит состояние приложения, работает только через сеттеры;
- Используется через наследование;
- Работает только с DOM.

### Класс CardCatalog
Назначение:
Отображает карточку товара в каталоге и инициирует действие выбора товара через внешний обработчик (actions).

Конструктор:
`constructor(container: HTMLElement, actions?: ICardActions)`

Поля:
- `categoryElement: HTMLElement` — DOM-элемент категории товара;
- `imageElement: HTMLImageElement` — DOM-элемент изображения товара.

Сеттеры:
- `category: string` — устанавливает текст категории и соответствующий CSS-класс в зависимости от значения. Использует categoryMap для установки CSS-классов категорий.
- `image: string` — устанавливает изображение товара через базовый метод setImage.

Обработчики:
- `onClick` — передаётся через actions и вызывается при клике на карточку, может содержать логику вида events.emit(...).

Особенности:
- Наследуется от ProductCard;
- Не использует IEvents напрямую;
- Не генерирует события внутри себя;
- Делегирует обработку клика через actions;
- Не содержит бизнес-логики;
- Работает только с DOM внутри своего контейнера;
- Использует ensureElement для поиска элементов.

### Класс PreviewCard
Назначение:
Отображает подробную информацию о товаре в модальном окне и позволяет инициировать добавление/удаление товара из корзины.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля:
- `button: HTMLButtonElement` — кнопка добавления/удаления товара;
- `_id: string` — внутренний идентификатор товара;
- `events: IEvents` — брокер событий для взаимодействия с презентером.

Сеттеры:
- `id: string` — устанавливает идентификатор товара (записывает во внутреннее поле _id);
- `inCart: boolean` — изменяет состояние кнопки (добавить / удалить из корзины).

События:
- `product:toggle` — генерируется при нажатии на кнопку, передаёт данные { id }.

Особенности:
- Наследуется от ProductCard;
- Не хранит состояние корзины;
- Не изменяет модели данных напрямую;
- Использует _id как внутреннее хранилище идентификатора;
- Только отображает состояние и сообщает о действиях пользователя;
- Использует ensureElement для поиска DOM-элементов.

### Класс BasketCard
Назначение:
Отображает товар в корзине и позволяет инициировать его удаление.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля:
- `indexElement: HTMLElement` — отображает порядковый номер товара;
- `deleteButton: HTMLButtonElement` — кнопка удаления товара из корзины;
- `_id: string` — идентификатор товара;
- `events: IEvents` — брокер событий.

Сеттеры:
- `id: string` — устанавливает идентификатор товара (записывает значение во внутреннее поле `_id`);
- `index: number` — устанавливает порядковый номер товара в списке.

События:
- `basket:remove` — генерируется при клике на кнопку удаления, передаёт { id }.

Особенности:
- Наследуется от ProductCard;
- Не управляет корзиной и её состоянием;
- Только инициирует удаление через событие;
- Не содержит бизнес-логики;
- Работает только с DOM внутри контейнера.

### Класс Basket
Назначение:
Отображает содержимое корзины, общую стоимость и управляет UI оформления заказа.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля:
- `listElement: HTMLElement` — контейнер списка товаров;
- `priceElement: HTMLElement` — отображение общей стоимости;
- `button: HTMLButtonElement` — кнопка оформления заказа;
- `events: IEvents` — брокер событий.

Сеттеры:
- `items: HTMLElement[]` — обновляет список товаров в корзине;
- `total: number` — устанавливает общую стоимость;
- `disabled: boolean` — управляет доступностью кнопки оформления заказа.

События:
- `order:start` — генерируется при нажатии на кнопку оформления заказа.

Особенности:
- Не хранит список товаров;
- Отображает только переданные данные;
- Не работает с моделями напрямую;
- Только инициирует переход к оформлению заказа.

### Класс Form
Назначение:
Базовый класс для всех форм. Обрабатывает ввод данных и уведомляет презентер об изменениях и отправке.

Конструктор:
`constructor(container: HTMLFormElement, events: IEvents)`

Поля:
- `submitButton: HTMLButtonElement` — кнопка отправки формы;
- `errorElement: HTMLElement` — элемент отображения ошибок;
- `events: IEvents` — брокер событий.

Сеттеры:
- `valid: boolean` — управляет доступностью кнопки отправки;
- `errors: string` — отображает текст ошибок валидации.

События:
- `form:submit` — отправка формы;
- `form:change` — изменение поля формы, передаёт { field, value }.

Особенности:
- Не выполняет бизнес-валидацию;
- Только сообщает о событиях ввода;
- Используется как базовый класс;
- Не хранит состояние формы.

### Класс OrderForm
Назначение:
Форма выбора способа оплаты и ввода адреса доставки.

Конструктор:
`constructor(container: HTMLFormElement, events: IEvents)`

Поля:
- `cardButton: HTMLButtonElement` — кнопка выбора оплаты картой;
- `cashButton: HTMLButtonElement` — кнопка выбора оплаты наличными;
- `events: IEvents` — брокер событий.

Методы:
- `protected setPayment(payment: 'card' | 'cash')` — устанавливает выбранный способ оплаты и обновляет UI.

События:
- `order:payment` — изменение способа оплаты { payment };
- `form:change` — изменение данных формы { field, value };
- `order:submit` — отправка первой формы оформления заказа.

Особенности:
Наследуется от Form;
Не хранит данные заказа;
Только инициирует события;
Не содержит логики обработки заказа;
Работает через брокер событий (IEvents);
Использует модификатор button_alt-active для выбранного способа оплаты;
Является первым шагом оформления заказа.

### Класс Success
Назначение:
Отображает сообщение об успешном оформлении заказа.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля:
- `closeButton: HTMLButtonElement` — кнопка закрытия окна;
- `descriptionElement: HTMLElement` — элемент с текстом результата заказа;
- `events: IEvents` — брокер событий.

Сеттеры:
- `total: number` — отображает итоговую сумму заказа.

События:
- `success:close` — генерируется при закрытии окна.

Особенности:
- Используется как самостоятельный UI-компонент, который может быть передан в Modal через render.;
- Не содержит бизнес-логики;
- Только отображает результат операции;
- Не взаимодействует с моделями напрямую;
- Закрытие управляется через событие в презентере.

Презентер
Назначение:
Презентер отвечает за связывание слоёв Model и View и реализует основную логику приложения.
В данном проекте презентер реализован в файле main.ts без выделения отдельного класса.

Обязанности презентера:
- подписка на события от моделей и представлений;
- вызов методов моделей при действиях пользователя;
- обновление представлений при изменении данных;
- управление отображением модальных окон.

Особенности:
- не генерирует события;
- не хранит данные;
- не содержит DOM-логики;
- работает только через модели и представления;
- обновляет интерфейс только при событиях от моделей или при открытии модальных окон.
